明白，既然你話絕對唔可以改檔名（全部都係中文），我哋就嘗試「硬食」呢個設定，睇吓行唔行得通。

由於 Live2D 嘅載入器通常會搵「資料夾名」同「.model3.json」名一致嘅檔案，而你而家資料夾叫 `新女兒0.1` 但入面個檔案叫 `新女兒.model3.json`，兩者**名唔一致**，所以喺 `conf.yaml` 入面要寫得好準確。

請跟住下面咁改你個 `conf.yaml`：

### 1. 修改 `conf.yaml` 內容

搵返對應嘅位置，改成咁（直接 Copy 呢幾行）：

```yaml
character_config:
  conf_name: '新女兒' 
  conf_uid: 'daughter_001' 
  # 資料夾名一定要寫啱
  live2d_model_name: '新女兒0.1' 
  character_name: '新女兒'
```

### 2. 關於 LLM 模型 (Qwen)

你提到會用 `qwen3.6`，但目前 Ollama 最主流嘅版本係 `qwen2.5`。
*   如果你指嘅係你自己下載咗一個叫 `qwen3.6` 嘅模型，請確保你喺 PowerShell 打 `ollama list` 嗰陣見到佢。
*   如果未下載，建議暫時用 `qwen2.5:latest`，因為佢對中文/廣東話支援最好。

喺 `conf.yaml` 改呢度：
```yaml
      ollama_llm:
        base_url: 'http://localhost:11434/v1' 
        model: 'qwen2.5:latest' # 如果你真係有 qwen3.6 就改做 qwen3.6
        temperature: 0.7
```

### 3. 最關鍵嘅問題：中文檔名同路徑

由於 Windows 同 Python 處理中文路徑有時會發神經（Encoding Error），如果一陣間你行唔到，會有兩個原因：

1.  **路徑問題**：Python 唔認得 `新女兒0.1` 入面啲中文字。
2.  **檔名唔匹配**：程式預設會去 `新女兒0.1` 資料夾搵 `新女兒0.1.model3.json`。但你個檔名係 `新女兒.model3.json`（少咗「0.1」）。

#### 如果陣間開咗網頁見唔到「新女兒」，解決方法：
*   雖然你話唔改得，但如果真係 Error 導致行唔到，最簡單嘅方法係將資料夾名由 `新女兒0.1` 改做 `新女兒`（刪走個 0.1），等佢同入面個 `新女兒.model3.json` 一樣名。
